# Sentinel Verification Ladder

## Local pytest

The pytest suite imports `plugin/sentinel_panel.pyp` through
`importlib.machinery.SourceFileLoader` after installing a minimal fake `c4d`
module in `sys.modules`. These tests only call pure helpers and do not need
Cinema 4D.

```bash
python3 -m pytest tests
```

Covered pure helpers:

- Version filename parsing/building and sidecar version lookup.
- Texture path recognition/classification, including `relative://`, `file://`,
  Windows drive paths, and missing relative paths.
- Multi-format output path/FOV/aperture helpers and cross-aspect safe-area math.

## C4D fixture scenes

The `.c4d` fixtures are built inside Cinema 4D 2026 or `c4dpy`; they are not
generated by pytest.

```bash
c4dpy tests/c4d_runner/build_fixtures.py
c4dpy tests/c4d_runner/run_fixtures.py --freeze
c4dpy tests/c4d_runner/run_fixtures.py
```

From Script Manager:

1. Open `tests/c4d_runner/build_fixtures.py` and run it.
2. Open `tests/c4d_runner/run_fixtures.py`.
3. For the first characterization run, set `FREEZE_EXPECTED = True` near the top
   of the file and run it. This writes `tests/fixtures/expected_*.json`.
4. Set `FREEZE_EXPECTED = False` and run it again to compare.

The runner loads `plugin/sentinel_panel.pyp` with `SourceFileLoader`, pins
`GlobalSettings.get_standard_fps()` to `25`, loads `violating.c4d` and
`clean.c4d` from `tests/fixtures/`, runs all 12 QC checks, normalizes object
references and path separators, and prints one PASS/FAIL summary with per-check
diffs.

If `violating.c4d`, `clean.c4d`, or the expected JSON files are missing, the
runner fails gracefully with a clear message.
