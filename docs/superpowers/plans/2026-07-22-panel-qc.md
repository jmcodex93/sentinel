**Goal:** The panel's QC section — "C refinada": FAIL cards → WARN cards → folded OK line, per-card contextual actions (Select/Fix/Info/Accept inline, no popups), Fix-all. Reuses the shared QC scoring + baseline + fixes engines; the native QC tab stays untouched.

**Architecture:** Spec `docs/superpowers/specs/2026-07-22-panel-qc-design.md`. New ops in `panel_ops.py` (`panel/qc` read-only over the shared scoring already there; `panel/qc/select`, `panel/qc/accept`, `panel/qc/fix_all` mutations reusing `_op_form_gate_submit`'s exact accept+apply_fixes shapes and the native Select selector). SPA `PanelQcSection` inside `PanelPage`, pure grouping/action logic in `panelQc.ts`. Refresh = existing stamp polling; accept/fix echo fresh `qc` back.

**Tech Stack:** established; no new deps.

## Global Constraints

- Branch `feat/panel-qc` off main. Baselines: pytest 690, vitest 60. Trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- Native QC panel UNTOUCHED (parallel strategy — retirement is 6.4). Only `panel_ops.py`/SPA change server-side.
- QC data = the SAME shared scoring path as Overview/Reports (`active_rules_for_doc → run_all_checks → compute_score → webbridge.qc_report_payload`); `panel/qc` must NOT run an extra check pass beyond that one call — polling reads the cache.
- Accept + fix_all reuse `_op_form_gate_submit`'s exact shapes (web_ops.py:364+): accept = one baseline acceptance per new violation of the check via `baseline`/`_run_quality_gate`'s loop, author+reason mandatory; fix_all = `fixes.apply_fixes` single undo via `_gate_fix_payload`. Every mutation invalidates `check_cache` and returns `{ok, stamp, qc}` (fresh embedded QC).
- `can_select`/`can_fix`/`fix_action_id`/severity come from `CHECK_REGISTRY` (qc/registry.py) — not invented.
- Fix respects the palette confirm contract (materials/fps destructive) + enabled/reason, same as Overview's QcCard.
- Design tokens only; FAIL `--color-status-fail`, WARN `--color-status-warn` (lighter tint), accent never state.
- Ops never raise past drain; accept with empty author/reason → `{ok: False, error}` inline.

### Task 1: Ops — `panel/qc` + select + accept + fix_all

**Interfaces:**
- `_op_panel_qc(payload)` (read-only): doc guard → the shared scoring (reuse the exact block from `_panel_qc_block` but return the FULL per-check list, not just top-3). Group by severity into `{score:{passed,total,disabled}, fail:[check], warn:[check], ok_count, disabled_count}`. `check = {id,label,severity,count,new,accepted,detail,can_select,can_fix,fix_action_id,accepted_all}` — severity/can_fix/fix_action_id from `CHECK_REGISTRY`; can_select from the registry's select capability; detail from `qc_report_payload`'s per-check detail; accepted_all = new==0 and accepted>0.
- `_op_panel_qc_select(payload)` `{check_id}`: doc guard → run the check's native scene-selection (find the selector the native Select button uses — grep panel.py for the QC select handlers / the registry's select action; reuse it, do NOT duplicate selection logic) → `c4d.EventAdd()` → `{ok, stamp}`. Unknown/​non-selectable check_id → `{ok: False, error}`.
- `_op_panel_qc_accept(payload)` `{check_id, author, reason}`: validate non-empty author+reason (`{ok: False, error: "author_required"/"reason_required"}`) → doc guard → accept the check's current new violations into baseline (COPY the accept branch of `_op_form_gate_submit`, restricted to one check_id) → invalidate `check_cache` → `{ok, stamp, qc: <fresh _op_panel_qc payload>}`.
- `_op_panel_qc_fix_all(payload)`: doc guard → `fixes.apply_fixes` over all currently-fixable checks in one undo (COPY `_op_form_gate_submit`'s fix_all branch via `_gate_fix_payload`) → invalidate cache → `{ok, stamp, qc}`.
- Register all four in `PANEL_OPS`.
