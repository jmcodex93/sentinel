Decisión: ¿cómo empaquetamos la capa HTML interactiva cuando llegue?

A

Vanilla HTML/JS (sin build)

Páginas HTML + CSS + JS plano servidas tal cual, como los informes. Cero toolchain, cero dependencias, cualquier editor las toca. Menos cómodo para UI muy dinámica (el futuro Hub), pero coherente con la filosofía stdlib del plugin.

B

SPA con build (Vite, como Overseer)

Toolchain moderno (componentes, reactividad). Mejor DX para pantallas complejas, pero introduce node/npm en un proyecto que hoy es 100% Python puro, y el build compilado es más opaco de mantener.
